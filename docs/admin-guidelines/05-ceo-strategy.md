# 비즈니스 전략 지침 (CEO)

## 비즈니스 핵심 KPIs (매일 확인)

**수익 지표**
- 일일 광고 노출/클릭/수익 (AdMob RPM, 총 eCPM)
- 광고 시청 완료율 (보상형 광고 기준)
- 유저 1인당 일평균 광고 시청 횟수

**참여 지표**
- DAU / MAU, DAU/MAU 비율 (스티키니스)
- 미션 완료율 (만보기/출석/광고/퀴즈/룰렛별)
- 티켓 발급량 vs 소비량 (재고 밸런스)
- 추첨 응모율 = 응모 유저 / DAU

**성장 지표**
- 신규 가입자 수 (카카오 로그인 기준)
- 레퍼럴 전환율 (초대 클릭 → 가입 → 첫 미션 완료)
- D1 / D7 / D30 리텐션

---

## 개발 우선순위 기준

| 우선순위 | 기능 | 이유 |
|----------|------|------|
| P0 | 실시간 대시보드 (DAU, 광고수익, 티켓 현황) | 운영 가시성 없이는 의사결정 불가 |
| P0 | 추첨 결과 관리 + 당첨자 발표 | 핵심 루프이자 법적 의무 |
| P1 | 유저 조회 + 정지/제재 도구 | 어뷰저 대응 없으면 광고수익 손실 |
| P1 | 레퍼럴 분석 (초대자→피초대자 전환) | 주요 성장 채널 |
| P2 | 미션/티켓 설정 변경 (서버 사이드) | 이벤트 유연성 |
| P2 | 상품 재고 및 예산 추적 | 비용 통제 |
| P3 | 공지사항/팝업 관리 | 운영 편의 |

---

## 운영 리스크 관리

- **티켓 어뷰징**: 1인 다계정, 봇 미션 완료 → 이상 티켓 발급 패턴 알림
- **추첨 조작 의혹**: 당첨 로직은 서버사이드 + 결과 로그 불변 저장 (감사 trail)
- **광고 클릭 어뷰징**: 단시간 과도한 광고 시청 유저 자동 플래깅
- **레퍼럴 어뷰징**: 자기 초대, 가상번호 가입 탐지
- **상품 초과 지급**: 당첨 처리 중복 방지 (멱등성 보장)

---

## 성장 관련 기능

- 레퍼럴 퍼널: 초대링크 발송 → 가입 → 첫 미션 → 리워드 지급 단계별 전환율
- 코호트 리텐션 테이블 (가입일 기준 D1/D7/D30)
- 미션별 이탈 포인트 분석 (어느 미션에서 유저가 그만두는가)
- 이벤트 A/B 효과 측정 (티켓 보너스 전후 DAU 비교)

---

## 비용 관리

- **Supabase**: 월간 DB 용량, API 호출 수, 실시간 구독 연결 수 모니터링
- **Firebase**: FCM 발송량, Crashlytics 에러율
- **상품 예산**: 월별 당첨 상품 원가 vs 광고수익 대비 지급률 (목표: 수익의 30~50% 환원)
- **예산 상한 알림**: 월 상품 지급 예산 초과 시 추첨 일시 중단 자동화

---

## 법적/컴플라이언스

- **경품 표시법**: 당첨자 공개 의무, 당첨 확률 고지 (어드민에서 확률 설정값 공개)
- **개인정보보호법**: 유저 데이터 열람 로그 기록, 탈퇴 요청 처리 이력
- **추첨 투명성**: 당첨 알고리즘 버전 기록, 시드값 보관 (감사 가능)
- **광고 정책(AdMob TOS)**: 강제 광고 시청 유도 패턴 금지 → 미션 설계 가이드라인

## 어드민 접근 권한 정책

| 역할 | 접근 범위 |
|------|----------|
| CEO / 대표 | 전체 읽기 + KPI 대시보드 |
| 운영팀 | 유저 관리, 공지, 추첨 결과 처리 |
| 개발팀 | 시스템 설정, 미션 파라미터, 로그 조회 |
| 재무팀 | 수익/비용 대시보드만 (읽기 전용) |
| 외부 감사 | 당첨 이력 + 추첨 로그만 (읽기 전용) |

**원칙**: 최소 권한 원칙(Least Privilege), 민감 액션(유저 제재, 당첨 취소)은 2인 승인 필요.

## P0 착수 기준

대시보드(DAU + 광고수익 + 티켓 현황) → 추첨 관리 → 유저 제재 도구 순으로 진행.
운영 가시성과 리스크 통제를 동시에 확보하는 것이 최우선.
